# Internal Documentation

This folder contains internal documentation for the SMB AI Command Platform. This documentation is intended for the development team and stakeholders only.

## Contents

### Architecture (`/architecture`)
Technical architecture decisions and system design documents.

- `system-overview.md` - High-level system architecture
- `data-flow.md` - Data flow diagrams and descriptions
- `adr/` - Architecture Decision Records

### Business (`/business`)
Business context, product requirements, and roadmap.

- `product-vision.md` - Product vision and goals
- `module-specs.md` - Detailed module specifications
- `competitor-analysis.md` - Market and competitor analysis

### Security (`/security`)
Security architecture and compliance documentation.

- `security-model.md` - Security architecture overview
- `data-privacy.md` - Data handling and privacy policies
- `compliance.md` - Compliance requirements and status

## Guidelines

1. **Keep it updated** - Update documentation when making significant changes
2. **Be specific** - Include diagrams, code snippets, and examples
3. **Review regularly** - Schedule quarterly documentation reviews
4. **Use templates** - Follow the established formats for consistency

## Access Control

This documentation should not be shared externally. Customer-facing documentation should go in `/docs`.
